# 事故、返工与缺陷复盘

> **用途**：沉淀项目历史事故、返工、漏测和联调问题，避免同类问题重复发生。
> **维护方**：测试负责人 / 开发负责人 / 模块 owner

## 1. 事故索引

| 编号 | 标题 | 发现时间 | 严重级别 | 状态 | 关联规则 |
|------|------|----------|----------|------|----------|
| INC-001 | `<标题>` | `YYYY-MM-DD` | `<P0/P1/P2/P3>` | `<待修复/已修复/已复盘>` | `<business-rules/decisions>` |

## 2. 复盘模板

```markdown
### INC-001: <标题>

- **发现时间**：YYYY-MM-DD HH:mm
- **发现渠道**：线上报警 / QA / 客户反馈 / 联调 / code review
- **影响范围**：用户、订单、数据、接口、任务或环境范围。
- **直接原因**：触发故障的直接技术原因。
- **根因**：为什么流程、设计、测试或代码没有提前发现。
- **修复 commit / PR**：<链接或 SHA>
- **验证证据**：测试命令、真实请求、日志、DB 记录或截图。
- **防复发措施**：
  - 文档规则：补充到哪个 `docs/sdd-context/*.md`
  - 代码防线：新增哪个测试、断言、告警或 guard
  - 流程防线：哪个阶段必须新增检查项
- **复盘人**：<name>
```

## 3. 常见问题分类

| 分类 | 典型表现 | Superflow 阶段加固 |
|------|----------|-------------------|
| 需求理解偏差 | 字段语义、状态流、边界条件理解错误 | clarify / docs |
| 源码影响面遗漏 | 只改一处，没有反查所有引用 | design |
| 数据库合同遗漏 | 代码依赖字段，但 SQL/migration 未更新 | docs / verify |
| mock-only 假通过 | 只跑 mock 单测，未跑真实入口 | implement / verify |
| 异步链路漏验 | MQ、定时任务、回调没有真实证据 | verify |
| 上下文漂移 | 长会话压缩后忘记原始约束 | handoff / guard |

## 4. 复盘回写要求

事故复盘完成后，至少回写一处长期规则：

- 业务规则问题：写入 [business-rules.md](./business-rules.md)。
- 架构或工程约定问题：写入 [decisions.md](./decisions.md)。
- 外部系统或联调问题：写入 [external-systems.md](./external-systems.md)。
- 测试策略问题：写入当前 OpenSpec change 的 `tests.md` 和 `test-report.md`。

只在聊天记录里解释，不算复盘完成。
