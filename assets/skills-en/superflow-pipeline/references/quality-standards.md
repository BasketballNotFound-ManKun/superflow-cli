# Quality Standards

## Non-Negotiables

- No missing required documents.
- No placeholder such as TBD, TODO, later, or to be generated.
- No API contract after implementation.
- No unit-only completion for integration work.
- No fallback behavior without owner decision.
- No stale handoff hash.
- No root aggregate doc edits by ordinary worker batches.

## Evidence Standard

Claims need commands, outputs, environment, timestamps where useful, and file or
log anchors. If evidence cannot be collected, mark Blocked or Partially verified.
