# OpenSpec Format

## Change Layout

- proposal.md
- api.md
- design.md
- tasks.md
- tests.md
- specs/<capability>/spec.md
- traceability-matrix.md
- review-checklist.md
- sdd-quality-gate.md
- test-report.md

## Spec Style

Use requirement IDs and scenarios. Each scenario must map to at least one test
case and one traceability row.

## Task Style

Tasks must be atomic, ordered, dependency-aware, and linked to requirement IDs.
